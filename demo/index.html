Methodology: Both engines receive identical JSON node trees and run the same flexbox algorithm (resolve → place → recurse). The JS baseline uses structuredClone to prevent cross-run contamination. The WASM engine parses JSON, solves layout, and serialises results entirely in C++. Timings are averaged over 10 warm runs after 2 discarded warm-up iterations. At small node counts (<1 k) the JS↔WASM boundary and JSON round-trip dominate; C++ wins decisively above ~3–5 k nodes where compute time outweighs fixed overhead.
